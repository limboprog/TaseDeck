## Что такое TaseDeck

TaseDeck — десктопное приложение для управления связями между AI-агентами и MCP-серверами: кто с кем работает, какие инструменты доступны и что реально вызывается в рантайме.

Обычно настройка MCP быстро превращается в хаос: несколько `mcp.json`, одни и те же серверы копируются между проектами, непонятно, какой агент что вызвал. TaseDeck собирает всё в одном локальном интерфейсе:

- поиск и установка MCP-серверов из официального реестра;
- настройка локальных и удалённых серверов, `stdio`, HTTP и OAuth;
- привязка серверов к конкретным проектам и агентам;
- сохранение исходного конфига агента как точки отката;
- экспорт записей в нативный формат конфигурации агента;
- просмотр реальных вызовов инструментов во вкладке Usage.

**Агенты остаются нативными, MCP-серверы — настраиваемыми, а TaseDeck управляет топологией между ними.**

## Интерфейс

### Топология проекта

Вкладка **Projects** показывает схему проекта: проект → агент → пресет → MCP-серверы. У каждого агента — свой набор серверов, свой пользовательский пресет и своя экспортированная конфигурация.

![Project topology](./docs/assets/project-topology.png)

Это не просто список: видно, какие агенты подключены, какой пресет у каждого, какие серверы активны, где добавить или убрать сервер.

### Журнал вызовов

Вкладка **Usage** пишет вызовы MCP-инструментов через проектный прокси. Видно, кто вызвал, какой сервер и инструмент, успех или ошибка, и что вернулось в ответ.

<p align="center">
  <a href="./docs/assets/usage-log.png">
    <img src="./docs/assets/usage-log.png" alt="Usage log" width="100%" />
  </a>
</p>

Фильтры по источнику, серверу, дате и результату помогают отлаживать агента, проверять пресеты и понимать, какие инструменты реально используются.

### Настройка запуска

У каждого MCP-сервера — редактируемый профиль запуска: команда, аргументы, переменные окружения, заголовки, транспорт, список инструментов и результаты тестов.

![Server runtime](./docs/assets/server-runtime.png)

Для локальных серверов TaseDeck хранит собранную команду и проверяет инициализацию и список tools из UI. Для удалённых — HTTP и OAuth. Переключатели инструментов открывают агенту только нужные tools.

### Реестр MCP

Вкладка **MCP** — установленные серверы и поиск в реестре: просмотр, установка, правка конфига запуска, обновление метаданных инструментов.

![MCP registry](./docs/assets/mcp-registry.png)

Список установленных серверов компактный, детали — в правой панели. Поиск, установка, тесты и правки в одном месте.

## Основные механики

### Обнаружение агентов и разбор конфигов

При первом запуске TaseDeck ищет конфиги известных агентов:

- Cursor
- Claude Code
- VS Code
- OpenCode
- Windsurf
- Codex CLI
- Antigravity
- GitHub Copilot

Для каждого типа известны путь к конфигу и формат файла. TaseDeck читает то, что уже есть, убирает свои служебные записи и сохраняет пользовательские MCP-серверы как исходный конфиг агента.

Если у агента уже есть `mcp.json`, приложение разбирает его в структуру проекта и пресета по умолчанию — не нужно всё вводить заново.

### Исходный конфиг

TaseDeck хранит снимок исходной MCP-конфигурации — базовое состояние для связи проекта и агента.

На этом можно безопасно экспериментировать:

- импортировать серверы из конфига проекта или агента;
- сделать пользовательский пресет для агента;
- добавлять, удалять и переопределять серверы в TaseDeck;
- экспортировать управляемые записи через прокси;
- сбросить агента к исходному состоянию.

Служебные записи TaseDeck отличимы от пользовательских и удаляются без порчи нативного конфига.

### Сброс агента

Сброс для агента в проекте:

- восстанавливает `mcp.json` из сохранённого исходника;
- удаляет пользовательский пресет и кэш агента;
- отвязывает агента от проекта;
- убирает служебные записи TaseDeck, оставляя пользовательский конфиг.

Нужен, когда настройка разрослась, топологию надо пересобрать или вернуть агента «как было».

### Разные агенты — разные серверы

В одном проекте агенты могут использовать разные MCP-серверы. Например:

- Cursor — `AppDeploy` и `Linear`;
- Claude Code — `context7`;
- другой агент — минимальный read-only пресет;
- у каждого свои env, аргументы, заголовки и включённые tools.

Связи проект–агент, пресеты, кэш пользовательских пресетов и переопределения конфига отображаются деревом: у каждой ветки агента — свой пресет и список серверов.

### Пресеты и переопределения

Пресет — набор серверов. У проекта может быть импортированный пресет по умолчанию; агент может ответвить свой пользовательский.

Переопределения хранятся на связи проект–агент и накладываются на серверы пресета: аргументы, env, заголовки, команда запуска, включённые/выключенные tools. Не нужно клонировать целый сервер ради мелкой правки.

### Установка из реестра

Поддерживаются удалённые и локальные серверы. TaseDeck разбирает метаданные пакета, транспорт, env, шаблоны команд и URL. Локальные npm-пакеты — через `npm install -g ...`, запуск — `npx -y ...` или `npm exec ...` по профилю пакета.

В собранном приложении расширяется окружение shell, чтобы из GUI находились `npm` и прочие tools (Homebrew, `nvm`, `fnm`, Volta, установщик Node).

### Проверка в UI

Из интерфейса: инициализация, список tools, пробные вызовы, ошибки, обновление метаданных. Статус подключения и настройки tools видны в UI.

## Как это работает

```text
┌─────────────────────────────────────────────────────────────┐
│ React UI                                                     │
│ MCP · Projects · Usage · Profile                            │
└──────────────────────────┬──────────────────────────────────┘
                           │ Tauri invoke()
┌──────────────────────────▼──────────────────────────────────┐
│ Rust backend                                                 │
│ SQLite · реестр · OAuth · синхронизация конфигов · экспорт    │
└──────────────────────────┬──────────────────────────────────┘
                           │
        ┌──────────────────┼──────────────────┐
        ▼                  ▼                  ▼
   SQLite              Папки проектов      MCP Registry
   пресеты, агенты     .cursor/mcp.json    registry.modelcontextprotocol.io
   серверы, usage      .tasedeck/proxy.mjs
```

### Установка из реестра

```text
Запись MCP Registry
  → разбор реестра
  → план установки
  → локальная установка или удалённый конфиг
  → строка mcp_servers в SQLite
  → обновление профиля запуска и tools
  → доступно в MCP и Projects
```

### Экспорт проекта

```text
Проект + агент + пресет
  → список серверов
  → переопределения агента
  → sidecar .tasedeck/mcp/{server}.json
  → копия .tasedeck/proxy.mjs
  → служебные записи TaseDeck в конфиге агента
  → агент запускает прокси как MCP-сервер
```

### Журнал вызовов

```text
Агент
  → запись в mcp.json проекта
  → .tasedeck/proxy.mjs
  → downstream MCP-сервер
  → ответ агенту
  → log spool
  → ingestor на Rust
  → usage_log в SQLite
  → вкладка Usage
```

## Конфигурация

TaseDeck не подменяет системы конфигов агентов — работает вместе с ними.

Проектные MCP-конфиги пишутся в нативное место для типа агента:

- `.cursor/mcp.json`
- `.vscode/mcp.json`
- аналоги для других агентов
- TOML для Codex CLI при необходимости

Сгенерированные записи указывают на проектный прокси TaseDeck. Вспомогательные файлы:

```text
.tasedeck/
  proxy.mjs
  mcp/
    server-name.json
```

Runtime-данные остаются в папке проекта; конфиг можно переносить вместе с проектом. TaseDeck переписывает только свои записи — пользовательские MCP-записи сохраняются.

## Хранение и безопасность

Данные приложения — в системной папке данных ОС. На macOS:

```text
~/Library/Application Support/TaseDeck/User/Storage/
```

В SQLite хранятся: MCP-серверы, конфиг из реестра, агенты, проекты, пресеты, переопределения, настройки tools, журнал вызовов.

Секреты шифруются до записи. Поддерживается OS Keychain и локальный master key как запасной вариант.

Удалённые MCP с OAuth — PKCE и callback/deep link `tasedeck://`. Access tokens отделены от статических значений в конфиге.

## Разработка

### Требования

| Инструмент | Версия |
|------------|--------|
| Node.js | 20+ |
| Rust | stable |
| Tauri | см. [документацию Tauri 2](https://v2.tauri.app/start/prerequisites/) |

### Запуск

```bash
git clone https://github.com/limboprog/TaseDeck.git
cd TaseDeck
npm install
npm run tauri dev
```

### Сборка

```bash
npm run tauri build
```

Установщики: `src-tauri/target/release/bundle/`  
Иконки генерируются из `src-tauri/icons/app-icon.svg`.
